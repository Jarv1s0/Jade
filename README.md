# Jade

**Jade** 是一款专为追求极致效率与审美体验的开发者设计的浏览器扩展。它完美融合了**高级书签维护工具**与**极简沉浸式新标签页**，旨在通过减少视觉干扰和自动化日常任务，提升您的数字生活品质。

---

## ✨ 核心特性

### 极简沉浸新标签页 (New Tab)
告别杂乱的导航，重拾纯粹的阅读与灵感之旅。
- **动态艺术壁纸**：支持 Bing 每日精选、NASA 天文影像、Picsum 随机摄影。
- **壁纸收藏系统**：一键收藏心动瞬间，支持收藏图片自动轮播。
- **无感交互设计**：
  - **隐藏式菜单**：主界面仅保留核心操作点，通过毛玻璃悬浮板切换源。
  - **沉浸式图说**：点击版权信息，深度阅读每一张壁纸背后的科学与人文故事。

### 高级书签工具箱 (Bookmark Toolbox)
让书签管理不再成为负担。Jade 提供了一系列自动化脚本，精准维护您的收藏。
- **清理与维护**：
  - **重复扫描**：深度比对 URL，清理冗余书签。
  - **失效探测**：检测链接存活状态；首次主动运行时会请求网站访问权限，拒绝后不会执行扫描。
  - **空文件夹清理**：一键移除逻辑混乱的嵌套空层级。
- **增强型数据操作**：
  - **无缝拖拽整理**：原生级拖拽体验，支持同级上下排序、直接拖入分类下级，以及拖入面包屑导航进行历史层级跃迁。
  - **未分类归档**：在“未分类书签”中可通过右键菜单将单个书签移动到任意真实文件夹。
  - **正则批量替换**：支持通过正则表达式批量修改书签链接（如批量切换域名或协议）。
  - **统计分析**：生成关于书签结构、深度及常用域名的可视化分析报告。
- **现代化极致 UI**：
  - 侧边栏与弹出窗无缝切换，支持全局快捷键 `⌘K` 启动搜索。
  - **无感视觉洁癖**：全局隐藏物理滚动条，极简“近期访问”重构，留白与对齐极致苛求。

## 🚀 安装指引

1. 克隆或下载本项目到本地。
2. 打开 Chrome 浏览器，进入 `chrome://extensions/`。
3. 开启右上角的 **开发者模式**。
4. 点击 **加载已解压的扩展程序**，选择本项目根目录。

## 📦 本地打包

打包前建议先执行本地验证：

```powershell
npm run verify
```

验证内容包括关键 JavaScript 语法检查、发布 workflow 版本来源检查、书签统计/清理纯函数检查，以及批量 URL 正则替换规则检查。

在项目根目录执行：

```powershell
.\release\build.ps1
```

脚本会自动读取 `manifest.json` 里的版本号，并在 `release/` 目录下生成类似 `Jade-v1.1.zip` 的安装包。
打包脚本会复用临时 staging 目录，并排除源码目录中的嵌套 `.zip` 文件，避免未使用素材进入扩展包。

如果你想自定义输出文件名：

```powershell
.\release\build.ps1 -OutputName Jade.zip
```

## 🔐 权限说明

- `bookmarks`：读取、展示、编辑、导入、导出和清理书签。
- `tabs`：打开书签、复用当前新标签页、打开 `chrome://bookmarks/` 与 `chrome://history/`。
- `favicon`：显示书签网站图标。
- `sessions`：展示最近关闭的标签页或窗口。
- `history`：展示最近访问记录。
- `sidePanel`：支持在浏览器侧边栏打开书签工具箱。
- `storage`：保存置顶书签、隐藏最近访问项、壁纸设置、收藏壁纸和 NASA API Key。
- `downloads`：导出书签 HTML 文件。
- `search`：新标签页搜索框调用浏览器默认搜索引擎。
- 可选网站访问权限：仅在您主动运行“失效链接检测”时申请，用于向书签中的 HTTP(S) 地址发起探测请求。

## ⚠️ 已知限制

- 失效链接检测受 CORS、反爬、登录态、网络波动影响；工具会把不确定结果单独标记，避免误删。
- NASA 默认 `DEMO_KEY` 有速率限制；可在新标签页设置自己的 API Key，配置只保存在本地浏览器。
- 批量 URL 替换使用 JavaScript 正则表达式，正式替换前必须先预览。
- 删除和批量 URL 替换无法在工具箱内撤销；如需保留副本，请先使用“导出书签”手动备份。
- `npm run verify` 覆盖语法、关键纯函数、权限声明和发布流程约束；发布工作流会在验证通过后另行执行打包。完整交互仍需在 Chrome 中加载未打包扩展后人工确认。

## 🎨 技术栈

- **Core**: 原生 HTML5 / CSS3 / JavaScript (ES6+)。
- **Zero Framework**: 不依赖任何 React/Vue 等框架，追求毫秒级的冷启动响应。
- **Performance**: 极轻量级的内存占用，后台脚本仅在任务执行时唤醒。
